Java: Kế thừa và đa hình

Trong Java, lập trình hướng đối tượng (OOP) cung cấp hai khái niệm quan trọng: kế thừa (inheritance) và đa hình (polymorphism). Chúng giúp tái sử dụng mã nguồn, mở rộng chương trình và linh hoạt trong thiết kế.

1. Kế thừa (Inheritance)

  • Định nghĩa: Kế thừa cho phép một lớp (subclass) sử dụng thuộc tính và phương thức của lớp khác (superclass).
  • Cú pháp:
class SuperClass {
    // Thuộc tính và phương thức
}

class SubClass extends SuperClass {
    // Thuộc tính và phương thức riêng
}

  • Lợi ích:

    • Tái sử dụng mã nguồn
    • Mở rộng chức năng mà không viết lại code

2. Abstract Class

  • Định nghĩa: Lớp abstract là lớp không thể khởi tạo trực tiếp, có thể chứa phương thức abstract chưa có thân, buộc subclass triển khai.
  • Ví dụ:
abstract class Animal {
    String name;
    abstract void makeSound(); // phương thức trừu tượng

    void eat() {
        System.out.println(name + " đang ăn.");
    }
}

class Dog extends Animal {
    void makeSound() {
        System.out.println(name + " sủa: Woof!");
    }
}

3. Interface

  • Định nghĩa: Interface định nghĩa các phương thức mà lớp implement phải triển khai. Java hỗ trợ đa kế thừa thông qua interface.
  • Ví dụ:
interface Flyable {
    void fly();
}

class Bird implements Flyable {
    public void fly() {
        System.out.println("Chim đang bay.");
    }
}

4. Đa hình (Polymorphism)

  • Định nghĩa: Đa hình cho phép một đối tượng có thể được tham chiếu bởi nhiều kiểu khác nhau.
  • Ví dụ:
public class Main {
    public static void main(String[] args) {
        Animal myDog = new Dog(); // tham chiếu đa hình
        myDog.name = "Lucky";
        myDog.makeSound(); // Gọi phương thức của Dog
        myDog.eat();

        Flyable myBird = new Bird();
        myBird.fly();
    }
}
  • Ở ví dụ trên, myDog là đối tượng Dog, nhưng được tham chiếu bằng kiểu Animal, đây là đa hình thời gian chạy.

5. Kết Luận:

  • extends: kế thừa từ lớp khác.
  • abstract class: lớp trừu tượng, không thể khởi tạo, chứa phương thức abstract.
  • interface: định nghĩa hợp đồng, lớp implement phải triển khai.
  • Polymorphism: cùng một đối tượng có thể hành xử khác nhau khi được tham chiếu bởi các kiểu cha hoặc interface.
